PROJECT OVERVIEW
----------------
TeamMatch is a cloud-native web application built for computer science instructors
to automatically form balanced, schedule-compatible project teams from student
rosters. The system collects detailed student skills, availability, and
preferences, then uses a deterministic optimization engine to generate fair,
explainable team assignments. Beyond initial matching, it supports ongoing team
collaboration through weekly check-ins, progress tracking, project milestones,
and AI-powered team health analysis.

================================================================================
MATCHING ENGINE & ALGORITHM
================================================================================

Location: teammatch/backend/app/core/matching.py

How it works:
  1. Team Formation
     - Seeds each team with one leader or flexible student
     - Greedily fills remaining slots, prioritizing students who add new
       skills not already represented on the team
     - Balances team sizes within the configured team_size parameter

  2. Scoring System (per team)
     - Skill Balance      (40% weight): ratio of unique skills to total skills
     - Experience Balance (35% weight): spread across beginner / intermediate / advanced
     - Schedule Overlap   (25% weight): pairwise availability intersection ratios
     - Overall Score = 0.4 * skill + 0.35 * experience + 0.25 * schedule

  3. Explanation Generator
     - Produces a human-readable summary for each team
     - Lists covered skills, experience mix, and schedule overlap percentage

  4. Execution
     - Runs asynchronously as a background task
     - Snapshots the student roster at match time
     - Creates new Team records in the database
     - Preserves old teams so historical check-in data remains intact
     - Atomic transactions with rollback on error
     - Auto-recovers stuck PENDING/RUNNING runs on server startup

API ROUTES (FastAPI)
================================================================================

Courses  (/courses)
  POST   /                      Create a new course
  GET    /                      List all courses
  GET    /code/{team_code}      Look up course by join code
  GET    /{course_id}           Get course details
  DELETE /{course_id}           Delete course (cascading)

Students  (/students)
  POST   /                      Create / enroll a student
  GET    /course/{course_id}    List students in a course
  GET    /{student_id}          Get student details

Match Runs  (/matchruns)
  POST   /                      Trigger the matching algorithm (background task)
  GET    /{run_id}              Get match run status
  GET    /course/{course_id}    List all match runs for a course

Teams  (/teams)
  GET    /course/{course_id}           Teams from latest match run
  GET    /matchrun/{match_run_id}      Teams from a specific run
  GET    /{team_id}                    Get team details
  POST   /{team_id}/analyze            AI-powered team health analysis

Check-ins  (/checkins)
  POST   /                      Submit a weekly check-in
  GET    /student/{student_id}  Student's check-in history
  GET    /team/{team_id}        Team's check-in history (newest first)
  GET    /course/{course_id}    Course-wide check-in history

Analytics  (/analytics)
  GET    /course/{course_id}    Aggregated team and student analytics

Projects  (/projects)
  POST   /                         Create a project
  GET    /course/{course_id}       Get course projects
  GET    /team/{team_id}           Get team projects
  PATCH  /{project_id}/assign      Assign project to a team
  POST   /{project_id}/milestones  Add a milestone

Notifications  (/notifications)
  POST   /                          Create a notification
  GET    /student/{student_id}      Get student notifications
  PATCH  /{notification_id}/read    Mark notification as read

================================================================================
AI INTEGRATION
================================================================================

Team Health Analysis uses the Anthropic Claude API:
  - Sends team composition, member skills, and check-in history to Claude
  - Returns a structured analysis including:
      * Overall team health summary
      * At-risk student identification
      * Recommended instructor action items
      * Week-by-week improvement suggestions
  - Triggered per-team from the /teams page or via POST /{team_id}/analyze

================================================================================
KEY DESIGN DECISIONS
================================================================================

1. Deterministic Matching
   Uses a greedy algorithm for reproducibility and explainability, making it
   easy for instructors to understand and justify team assignments.

2. Async Processing with Recovery
   Match runs execute as background tasks with status polling. On server
   restart, stuck PENDING or RUNNING jobs are automatically recovered.

3. Team Immutability After Re-matching
   Old teams are preserved when a new match run is triggered, so all
   historical check-in data remains valid and accessible.

4. AI for Qualitative Insights
   The matching algorithm handles objective scoring; Claude handles
   subjective, contextual interpretation of team health and student risk.

5. Local Storage Authentication
   Role and identity management via local storage keeps the system simple
   for a course context without requiring full OAuth infrastructure.

6. Cloud-Native Azure Architecture
   Managed services (PostgreSQL, App Service, Static Web Apps) reduce
   operational overhead while providing production-grade reliability.
